# 色彩設計系統

**之翰の備忘錄 · DOC.01 / COLOR · REV 2026.05**

> 這份色彩系統服務於一個編輯式的技術期刊。它的目標不是視覺上的活潑，而是讓長時間的閱讀仍然舒適。所有色彩被設計成在白天與印刷品之間都能成立。

---

## § 01 · 主色版（Core Palette）

八個核心色票。前四個為「紙」——構成所有底層；中間三個為「墨」——承擔所有文字；最後兩個為「青藍」，唯一的彩色。

| 編號 | 名稱 | Hex | Token |
|------|------|-----|-------|
| N°.01 | 紙白 Paper | `#F7F3EE` | `--bg` |
| N°.02 | 米白 | `#F1ECE4` | `--bg-soft` |
| N°.03 | 純白 | `#FFFFFF` | `--surface` |
| N°.04 | 界線 | `#E6E0D8` | `--border` |
| N°.05 | 墨黑 Ink | `#1F1F1F` | `--text` / `--rule` |
| N°.06 | 淡墨 | `#6B6B6B` | `--text-muted` |
| N°.07 | 霧墨 | `#9A9A9A` | `--text-soft` |
| N°.08 | 青藍 Accent | `#4A6FA5` | `--accent` |
| N°.09 | 深青 | `#2F4D78` | `--accent-ink` |

## § 02 · Token 命名與職責

每個 token 以「它做什麼」命名，而非「它長什麼樣」。

### Paper · 底層

| Token | Hex | 用途 |
|-------|-----|------|
| `--bg` | `#F7F3EE` | 頁面預設底色。所有區塊的起點 |
| `--bg-soft` | `#F1ECE4` | 替換式區塊（頁尾、section--soft）提供節奏感 |
| `--surface` | `#FFFFFF` | 卡片、文章內頁、浮起層 |
| `--border` | `#E6E0D8` | 所有實線分隔、卡片邊框 |
| `--border-soft` | `#EDE7DD` | 較輕的分隔（背景與物件接近時） |

### Ink · 文字

| Token | Hex | 對比 | 用途 |
|-------|-----|------|------|
| `--text` | `#1F1F1F` | 14.2:1 | 所有正文、標題主色 |
| `--text-muted` | `#6B6B6B` | 5.6:1 | 摘要、副標、解說 |
| `--text-soft` | `#9A9A9A` | 3.1:1 | 日期、編號、元資料（僅大字級） |
| `--rule` | `#1F1F1F` | — | 章節分隔粗線（與 `--text` 同色，語意不同） |

### Accent · 唯一彩色

| Token | Hex | 對比 | 用途 |
|-------|-----|------|------|
| `--accent` | `#4A6FA5` | 4.8:1 | 連結、英文斜體裝飾、首字放大、引述邊條 |
| `--accent-ink` | `#2F4D78` | 8.6:1 | hover 狀態、內嵌程式碼文字 |
| `--accent / 18%` | `rgba(74,111,165,.18)` | — | 文字選取背景 |

## § 03 · 設計哲學

### i. 紙是主角，墨是配角，藍是標點

一個編輯式網站的色彩比重不該與一個 SaaS 控制台相同。  
用 warm 的米色紙底取代過於冷冽的純白；用一個近乎全黑的 ink 承擔所有文字；只用一個藍色點綴，並且嚴格控制它的出現頻率——它是頁面上最稀有的東西。

**我們不接受第二個彩色。** 如果一個訊息需要被注意到，應透過排版、留白、或位置達成，而非再加一個顏色。

### ii. 對比是骨架，不是裝飾

所有色票圍繞著一條 contrast 軸線設計——從 `--bg`（L 95）到 `--text`（L 14），中間留下三個刻度給次要文字。這條軸線決定了閱讀的階層；我們不靠粗體與字級，而是靠灰階。

三個灰——不多也不少。多了稀釋階層，少了壓扁版面。

### iii. 為印刷品而生

這份色票經過 CMYK 試印測試，所有顏色都能在 100lpi 的紙本上重現：
- 沒有極端飽和
- 沒有 web-only 的 neon
- 沒有純黑（純黑在油墨上吸光，反而失去重量感）

同一份系統可以同時印在 80gsm 內頁、200gsm 封面、與 Retina 螢幕上。

### 不在系統裡的東西

- 沒有 success / warning / error 狀態色（個人期刊不需要，未來獨立為 signal 子系統）
- 沒有漸層
- 沒有陰影色票（陰影用半透明純黑，與底色無關）

系統提供的反白組合，目前用於 `.prose pre`。並非完整的暗色主題——僅讓 monospace 字符在長段裡更舒適。

|------|------|

| Default text | `#F1ECE4` |
| Comment | `rgba(241,236,228,.55)` |
| Keyword | `#8FA5C6` |
| String | `#B7C5DD` |

## § 07 · 使用準則

### ○ DO · 鼓勵

1. 用 `--bg` 為起點，所有其他色都是它的對照
2. 用三階灰建立階層；少於三階不夠，多於三階就太花
3. 讓 `--accent` 保持稀有——它出現的次數應該可以數得出來
4. 用 token 名稱（如 `--text-muted`）討論，不用 hex 值

### × DON'T · 避免

1. 不要新增第二個彩色。若需要區分，請用排版或位置而非顏色
2. 不要把 `--accent` 拿去做按鈕底色。它是「指向」的顏色，不是「表面」的顏色
3. 不要用 `--text-soft` 作為正文色——它的對比是為小級數元資料設計的
4. 不要使用漸層、霓虹色、或飽和度高於 60% 的色彩。它們不能在紙上印出來
